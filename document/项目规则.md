# HypnoVibe 项目规则

> 最后更新：Phase 2 完成时  
> 适用：本项目所有开发阶段的文档与代码操作

---

## 1. 文档编辑规则

### 禁止

- **禁止使用 PowerShell 或命令行工具修改 `.md` 文档**，包括但不限于：
  - `Set-Content`
  - `Out-File`
  - `Get-Content | ... | Set-Content`
  - `>` 重定向、`>>` 追加
  - 任何涉及文件编码转换的 shell 命令

> 原因：PowerShell 的默认编码行为会破坏 UTF-8 中文文档，且破坏不可逆。曾因此导致 4 份设计文档全部损坏，耗费大量时间重构。

### 允许

- **文档创建**：`Write` 工具（UTF-8 安全）
- **文档修改**：`SearchReplace` 工具（精确行级替换）
- **文档批量替换**：每个文件单独调用 `SearchReplace`，不允许批量 shell 替换
- **文档读取**：`Read` 工具

### 代码文件

- 代码文件（`.kt` / `.java` / `.xml` / `.kts` / `.proto` / `.cmake`）无编码风险，可用 `Write` / `SearchReplace` / shell 均可

---

## 2. 构建规则

- 编译命令：`./gradlew assembleDebug`（需先设置 `$env:JAVA_HOME`）
- 签名密钥：`hypnovibe-release.jks`，密码 `hypnovibe2026`，alias `hypnovibe`
- 密钥配置文件：`keystore.properties`（已在 `.gitignore` 排除）

---

## 3. 文件扩展名约定

| 类型 | 扩展名 | 格式 |
|------|--------|------|
| 设备配置 | `.hvconfig` | JSON |
| 时间轴脚本 | `.hvscript` | JSON + Base64(Protobuf) |
| 播放列表 | `.hvplaylist` | JSON |
| 波形预设 | `.hvwav` | JSON + Base64(Protobuf) |

所有数据文件存储在 `{context.filesDir}/` 下对应子目录。
