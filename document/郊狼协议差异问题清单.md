## 问题详情

### 问题 1：DeviceProtocolAdapter 单通道接口不兼容 B0 双通道合并指令

**涉及位置**：[功能设计文档 §9.7](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L735-L751)

**当前设计**：

```java
fun encodeCommand(channelId: String, strength: Int): ByteArray
```

**协议实际**：郊狼 V3 的 B0 指令是 **20 字节的 A+B 双通道合并指令**，每 100ms 发一条。无法通过单通道单次调用的接口来构造。

**建议修改**：

```java
// 改为批量编码接口
fun encodeCycleCommand(
    channelStrengths: Map<String, Int>,
    channelWaveforms: Map<String, WaveSlot[]>
): ByteArray
```

其中 `WaveSlot` 同时包含频率和强度值（参见问题 2）。

### 问题 2：WaveformFile 缺少频率维度

**涉及位置**：[功能设计文档 §4.3.2](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L182-L198)

**当前设计**：

```json
"dataPoints": [{ "timeMs": 0, "strength": 10 }]
```

仅含强度值。但郊狼 B0 指令每 25ms 槽位需要**频率 + 强度**两个维度。DG-LAB 内置的"呼吸""潮汐"波形都是频率和强度同时变化的。按当前模型导入郊狼波形后，频率信息会丢失。

**建议**：二选一

- **方案 A**：扩展 `dataPoints` 增加 `frequency` 字段（通用性强）
- **方案 B**：为郊狼格式单独保留 `rawFrames` 字段（保持 dataPoints 简洁）

### 问题 3：指令节流参数 50ms 与协议 100ms 固定周期冲突

**涉及位置**：[功能设计文档 §6.5](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L383-L390)

**当前设计**：

```
最小发送间隔：50ms
强度变化阈值：≥ 2
在设备类型级别可配置
```

**协议实际**：郊狼 V3 输出窗口固定 100ms，B0 指令只能每 100ms 发一次。50ms 间隔对郊狼无意义——发了也不会更快生效。

**建议**：将节流参数从"全局可配置"改为"每个设备类型硬编码"，郊狼固定 100ms，不可配置。将 `最小发送间隔` 改为 `输出窗口` 概念。

---

### 问题 4：设计文档未提及 BF 软上限指令

**涉及位置**：功能设计文档全文

**现状**：设计中未提及软上限、频率平衡参数等概念。但协议要求**每次重连后必须写入 BF 指令**，否则软上限可能是其他 APP 设置的未知值。

**建议**：在 CoyoteAdapter / CoyoteDevice 的初始化流程中明确加入 BF 写入步骤。可选地在 DeviceConfig 中增加 `softLimit` 字段供用户配置。

---

### 问题 5：TimelineEngine 输出仅含强度、缺少频率

**涉及位置**：[架构设计文档 §5.1](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/架构设计文档.md#L288-L291)

**当前设计**：

```
TimelineCalc (C) → 强度快照 Map<ch, Int>
```

**协议实际**：郊狼需要频率+强度两个维度，每 100ms 输出 4 组 (25ms×4)。

**建议**：将输出从 `Map<ch, Int>` 扩展为 `Map<ch, Slot[4]>`，每个 Slot = {frequency, strength}。Native 层需要感知设备输出窗口的概念。

### 问题 6：强度变化阈值 ≥2 粒度过粗

**涉及位置**：[功能设计文档 §6.5](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L383-L390)

**当前设计**：强度变化 < 2 不发送新指令。

**协议实际**：B0 指令中波形强度范围是 0-100，粒度已经是每步 1 单位。阈值设 ≥2 会吃掉微小变化，在低频低强度场景下可能影响体验。

**建议**：郊狼的强度变化阈值设为 1（有变化就发），反正 100ms 周期本身就是节流。

### 问题 7："保持心跳"描述不准确

**涉及位置**：[功能设计文档 §9.7](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L754-L758)

**当前描述**：

> 需要持续发送心跳/保活指令维持连接

**协议实际**：郊狼没有独立的心跳指令。B0 指令本身（每 100ms 发送）就是保活机制——如果不发 B0，设备不会断开 BLE，但波形停止输出。断开连接是由 BLE 协议层的 Supervision Timeout 决定的。

**建议**：修改描述为"需要每 100ms 持续发送 B0 指令以维持波形输出；停止发送则波形中断，但 BLE 连接保持"。

### 问题 8：设备类型 `coyote` 未区分 V2/V3

**涉及位置**：[功能设计文档 §2.1](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L60-L65)

**当前设计**：

```
coyote | 郊狼 DG-LAB | A 通道 + B 通道 | 0-200
```

**协议实际**：V2 和 V3 的蓝牙 UUID、指令格式、强度范围完全不同：
- V2: UUID `955A...`, 强度 0-2047 (S值), 分离的 PWM_AB2/A34/B34 特性
- V3: UUID `0000...`, 强度 0-200, 合并的 B0 指令

**建议**：拆分为 `coyote_v2` 和 `coyote_v3` 两个设备类型，或者 `coyote` 下通过扫描到的蓝牙名称自动判断版本并加载对应适配器。

### 问题 9：波形强度映射描述有误

**涉及位置**：[功能设计文档 §4.3.3](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L208)

**当前描述**：

> 将 DG-LAB 的 0-1000 内部值映射到 0-200 外部强度

**协议实际**：V3 协议中没有 0-1000 的内部值。通道强度范围 0-200，波形强度范围 0-100。可能混淆了 V2 的强度 S 值（0-2047，但也不是 0-1000）。

**建议**：修正为准确的映射关系——V3 通道强度 0-200、V3 波形强度 0-100、V2 强度 S 值 0-2047。

### 问题 10：郊狼波形导入未明确频率处理

**涉及位置**：[功能设计文档 §4.3.3](file:///c:/Users/xyy/AndroidStudioProjects/HypnoVibe/document/功能设计文档.md#L200-L209)

**当前描述**：提到"解析 DG-LAB 官方 APP 的波形导出文件""脉冲模式"，但未说明波形文件包含频率数据以及如何处理。

**建议**：补充说明郊狼波形文件同时包含频率和强度两个维度，导入时需要保留两者。详见问题 2 的 WaveformFile 扩展方案。
