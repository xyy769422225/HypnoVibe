# 郊狼波形集成 — 可行性分析与实施计划

> 版本：v1.0
> 目标：将 .pulse 波形导入 + 内置官方波形 合并入郊狼测试面板，并打通波形到打轴的链路
> 涉及模块：设备管理、波形管理、打轴编辑器、播放引擎

---

## 一、可行性分析

### 1.1 郊狼测试面板集成波形 — ✅ 完全可行

**改动点**：
- `CoyoteTestScreen` 目前只有 A/B 通道手动强度调节
- 新增一个 tab/分段：手动模式 vs 波形模式
- 波形模式下选择波形文件（内置或导入），启动 100ms 定时器循环播放
- 定时器每 100ms 从波形数据取一帧，调用 CoyoteController 写入设备

**现有基础**：
- `CoyoteController.setManualStrength(a,b)` 已就绪
- `DGLabV3Adapter` 内部 100ms 定时器已就绪，只需改为接收波形帧而非手动强度
- 需要新增 `CoyoteController.sendWaveformFrame(frequencyA, strengthA, frequencyB, strengthB)` 方法

### 1.2 .pulse 文件导入 — ✅ 完全可行

**改动点**：
- 新建 `PulseFileParser` 工具类，解析 `Dungeonlab+pulse:` 格式
- 文件内容 → `PulseData` 数据结构（基于前一文档定义的模型）
- 导入后反序列化存储为 JSON（复用 `JsonFileRepository`）
- UI：点击导入 → 文件选择器（`GetContent` / `ActivityResultContracts.GetContent()`） → 调用解析器 → 存入 `waveforms/` 目录

**现有基础**：
- `.pulse` 格式分析文档已完成
- `JsonFileRepository` 通用持久化已就绪
- 官方 APP 的 `analysisPulse0()` 逻辑已逆向，可直译

### 1.3 内置官方波形 — ✅ 完全可行

**改动点**：
- 将官方常用波形（呼吸、潮汐等）硬编码为 `.pulse` 字符串常量
- App 首次启动时，通过 `WaveformRepository.initBuiltin()` 将内置波形写入文件系统
- 内置波形和导入波形合并显示在波形列表中

**现有基础**：
- 官方 APP 的 `v3/example.md` 包含呼吸、潮汐两套波形的 V3 频率/强度序列
- 可转换为 .pulse 格式的 section+points 表达并内嵌

### 1.4 波形用于打轴 — ✅ 完全可行，需少量适配

**改动点**：
- `TimelineScript.Keyframe` 新增 `waveformRef: String?` 字段（引用波形文件 ID）
- `KeyframeExpander.expand()` 增加 "waveform" 效果类型：不从函数计算关键帧，而是从 `.pulse` 文件直接读取帧序列
- `TimelineEngine.ChannelRuntime` 增加波形帧缓冲（按需加载）
- `PlaySessionVM` 的 positionPoller 中，对标记了 `waveformRef` 的关键帧，从波形数据中取帧而非从计算函数取

**现有基础（非常有利）**：
- `WaveformFile` 实体已定义（`waveformId`、`name`、`totalDurationMs` 等）
- `WaveformSegment` 实体已定义（含 `waveformRef`、`startTimeMs`、`endTimeMs` 等字段）
- `IWaveformRepository` 接口已定义，只需补充实现
- `Keyframe` 有 `waveMode: String` 字段，可新增 `"customWave"` 取值
- `TimelineEngine.ChannelRuntime` 已有 `freqIndex[]`、`strengthIndex[]` 并行数组，波形帧可直接填入

**唯一缺失**：`WaveformRepository` 的实现类和 `WaveformSegment` 到 `Keyframe` 的展开逻辑。这两部分都已有实体和接口定义，只需填空。

## 二、架构设计

### 2.1 数据层

```
PulseData (.pulse 内存模型)
  ├── L, playRate, ZY
  └── List<PulseSection>
       ├── A, B, J, PC, JIE
       └── List<PulsePoint> (y, anchor)

↓ 导入时

WaveformFile (持久化 JSON)
  ├── waveformId (UUID)
  ├── name
  ├── deviceType ("dglab_v3")
  ├── sourceFormat ("dglab_pulse_type0")
  ├── totalDurationMs
  └── sections: JSON (PulseSection[] 序列化)

↓ 打轴引用时

TimelineScript.Keyframe
  ├── timeMs
  ├── strength (百分比)
  ├── freq
  ├── waveMode ("customWave" 表示引用外部波形)
  └── waveformRef: String? (波形文件 ID)
```

## 四、风险与注意事项

| 项目 | 说明 |
|------|------|
| **频率数组** | `PulseToFrames` 的渐变插值需要参照官方 `classicP0` 的对数/线性/指数公式，否则体感与官方 APP 不对齐 |
| **V2/V3 差异** | V2 波形帧使用 `X+Y=freq, Z=strength/5` 公式；V3 直接用频率+强度 4槽位构造 B0。`PulseToFrames` 需区分两个版本 |
| **性能** | 长波形（3分钟+）展开后有 1800+ 帧，需懒加载/按需计算，避免预加载时 OOM |
| **兼容性** | 当前 `.pulse` 格式分析仅覆盖 pulseType=0（主格式）。pulseType=2 的 CSV 格式暂不处理（实测样本较少） |
| **WaveformFile 容量** | `.pulse` 文件通常 5-50KB，存为 JSON + Base64 不会超过 100KB，安全 |
