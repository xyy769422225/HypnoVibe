## 1. 系统概述

### 1.1 系统定位

HypnoVibe 是一个面向催眠音声爱好者的蓝牙玩具控制 APP。用户导入音频文件，为其编写/关联时间轴脚本，播放音频时 APP 根据时间轴实时向连接的蓝牙玩具发送控制指令。支持郊狼（Coyote DG-LAB）及其他成人蓝牙玩具。

### 1.2 核心工作流

```
设备配置定义 → 时间轴脚本制作（引用配置ID） → 创建播放列表（匹配配置ID） → 通道映射设置 → 开始播放 → 时间轴引擎实时驱动设备
```

### 1.3 模块总览

| 模块 | 职责 |
|------|------|
| 设备配置系统 | 定义通道名称/设备/强度范围，拥有唯一ID供时间轴引用 |
| 时间轴脚本系统 | 引用配置ID + 波形段组织 + 导入波形管理 |
| 波形文件系统 | 导入/存储/管理波形数据文件，支持郊狼特有格式 |
| 音频播放引擎 | 自研 MP3/WAV 播放，毫秒级时间回调 |
| 时间轴执行引擎 | 播放时根据时间轴实时发送指令 |
| 播放列表系统 | 音频+时间轴的组织、通道映射、播放模式 |
| 打轴编辑器 | 可视化时间轴制作，波形预设拖拽 |
| 设备管理 | 蓝牙扫描/连接/历史记忆/一键重连 |

## 2. 核心概念定义

### 2.1 设备类型（DeviceType）

系统支持可扩展的设备类型体系，每个类型定义自己的通道结构：

| 类型ID | 名称 | 通道结构 | 说明 |
|--------|------|----------|------|
| `coyote_v3` | 郊狼 DG-LAB v3 | A 通道 + B 通道 | 两个独立输出通道，各有强度 0-200 |
| `coyote_v2` | 郊狼 DG-LAB v2 | A 通道 + B 通道 | 旧版协议，强度范围可能不同 |
| `lovense_vibrate` | Lovense 震动类 | 震动通道 | 强度 0-20 |
| `generic_pwm` | 通用 PWM 玩具 | 单通道 | 强度 0-100 |
| *（可扩展）* | 后续新增类型 | 按需定义 | — |

### 2.2 通道（Channel）

- 每个设备类型有固定数量的**物理通道**（如郊狼固定有 A、B 两个输出通道）
- 通道是时间轴操作的最小控制单元
- 通道物理属性（强度范围）由设备类型决定

### 2.3 设备配置（DeviceConfig）

- **配置本身拥有唯一 ID**，是所有时间轴脚本引用配置的依据
- 配置中**定义各通道的完整信息**：通道 ID、通道名称（如"后穴"）、设备类型（如 `coyote_v3`）、强度范围
- 配置**不规定通道对应哪个物理通道（A/B）**——物理通道映射在播放时通过设备关联界面设置
- 用户为每个通道命名，让使用者知道该通道对应身体的哪个部位

### 2.4 时间轴脚本（TimelineScript）

- 时间轴脚本**引用一个配置 ID**，使用该配置定义的通道
- 时间轴文件内**不再内嵌通道定义**，只记录 `configId`
- 包含多个**波形段**（WaveformSegment），每个波形段作用于配置中定义的某个通道
- 一个音频可以关联 0 或 1 个时间轴脚本

### 2.5 播放列表（Playlist）

- 有序的**音频-时间轴**对
- **只有引用同一配置 ID 的时间轴脚本才能加入同一个播放列表**，保证各设备通道在多个时间轴文件中一致
- 含**通道映射表**：配置中定义的通道 → 物理设备通道
- 支持三种播放模式

## 3. 模块一：设备配置系统

### 3.1 功能描述

设备配置拥有**唯一 ID**，定义本次控制场景中所有通道：通道 ID、通道名称（如"后穴""前部"）、设备类型（如 `coyote_v3`）、强度范围。配置**不指定物理通道映射**——哪个物理设备通道（A/B）对接哪个配置通道，在播放时通过设备关联界面确定。时间轴脚本通过引用配置 ID 来保证通道定义的一致性。

### 3.2 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| DC-01 | 创建配置 | 创建新配置，定义通道（ID/名称/设备类型/强度范围） | P0 |
| DC-02 | 编辑配置 | 修改通道名称、增删通道、调整强度范围 | P1 |
| DC-03 | 删除配置 | 删除不再使用的配置，检查是否被时间轴引用 | P1 |
| DC-04 | 配置列表 | 查看所有已保存的配置 | P0 |
| DC-05 | 配置导出/导入 | JSON 格式导出、导入配置文件 | P2 |
| DC-06 | 配置克隆 | 基于现有配置创建副本（生成新ID） | P2 |

### 3.3 配置数据结构

```
DeviceConfig {
  id: String                    // 配置唯一ID（时间轴文件通过此ID引用）
  name: String                  // 用户命名的配置名称（如"双郊狼配置"）
  channels: [                   // 通道定义列表
    {
      channelId: String         // 通道唯一ID（如 "ch_1"）
      channelName: String       // 用户命名的通道名称（如 "后穴"）
      deviceType: String        // 设备类型（如 "coyote_v3"）
      minStrength: Int          // 最小强度
      maxStrength: Int          // 最大强度
      defaultStrength: Int      // 默认强度
    }
  ]
  createdAt: Long
  updatedAt: Long
}
```

### 3.4 配置的作用

- **配置 ID 是一致性保证**：时间轴脚本记录配置 ID，播放列表只允许相同配置 ID 的脚本加入
- **通道定义一次，多处复用**：多个时间轴脚本可引用同一配置，通道名称/类型保持一致
- 配置 = "有哪些控制通道、叫什么名字、属于哪种设备类型"
- **物理通道映射后置**：播放时用户决定每个配置通道接到哪个物理设备的哪个通道（A/B）

## 4. 模块二：时间轴脚本系统

### 4.1 功能描述

时间轴脚本**引用一个配置 ID**，使用该配置定义的通道。脚本主体是波形段数据，每个波形段引用一段导入的波形文件并指定作用于配置中的哪个通道。

### 4.2 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| TS-01 | 创建时间轴脚本 | 新建脚本，选择引用的配置 ID | P0 |
| TS-02 | 加载时间轴脚本 | 从文件读取并解析，校验 configId 有效性 | P0 |
| TS-03 | 保存时间轴脚本 | 将编辑结果写入文件 | P0 |
| TS-04 | 导入波形文件 | 导入郊狼或其他设备的波形文件作为波形段 | P0 |
| TS-05 | 添加波形段 | 将导入的波形放置到配置中某通道的时间位置 | P0 |
| TS-06 | 编辑波形段 | 调整起止时间、选择波形文件、编辑标签 | P0 |
| TS-07 | 删除波形段 | 删除选中的波形段 | P0 |
| TS-08 | 波形段标签 | 为每段波形添加/编辑标签文本 | P0 |
| TS-09 | 标签搜索/过滤 | 按标签快速定位波形段 | P1 |
| TS-10 | 波形段复制粘贴 | 复制波形段到同一/其他通道 | P1 |
| TS-11 | 通道禁用标记 | 时间轴中标记某通道为禁用（调试用） | P0 |
| TS-12 | 时间轴校验 | 检查波形引用完整性、ConfigId 一致性等 | P1 |
| TS-13 | 波形预设管理 | 管理已导入的波形文件，作为预设复用 | P0 |

### 4.3 波形系统

#### 4.3.1 设计原则

波形**不是简单数学函数**（非正弦/方波等），而是以**导入的波形数据文件**为基本单位。波形来源于：
- 郊狼 DG-LAB 官方 APP 导出的波形文件转换
- 其他设备厂商的波形格式转换
- 第三方波形数据

#### 4.3.2 波形文件格式

每个波形文件是一个独立的强度-时间数据序列：

```
WaveformFile {
  waveformId: String            // 唯一标识
  name: String                  // 波形名称（如 "呼吸感_慢"）
  deviceType: String            // 适用的设备类型（如 "coyote_v3"）
  sourceFormat: String          // 来源格式（如 "dg_lab_v2", "custom_csv"）
  totalDurationMs: Long         // 波形总时长
  dataPoints: [                 // 强度采样点数组
    {
      timeMs: Long              // 相对时间（毫秒）
      frequency: Int            // 频率值（郊狼特有）
      strength: Int             // 强度值
    }
    // ...
  ]
  sampleRate: Int               // 采样率（数据点/秒，如 20 表示每 50ms 一个点）
  tags: [String]                // 波形标签（如 "慢", "呼吸", "持续"）
}
```

#### 4.3.3 郊狼特有波形

郊狼 DG-LAB 设备支持其特有的波形指令格式，系统需要：

| 郊狼波形特性 | 说明 |
|-------------|------|
| DG-LAB 波形文件导入 | 解析 DG-LAB 官方 APP 的波形导出文件 |
| 脉冲模式 | 郊狼支持特定的脉冲序列模式 |
| 波形强度映射 | 将 DG-LAB 的 0-1000 内部值映射到 0-200 外部强度 |
| 波形循环标记 | 郊狼波形可标记是否循环播放 |

#### 4.3.4 波形预设系统

```
WaveformPresetLibrary {
  presets: [WaveformFile]       // 已导入的波形文件集合
  categories: [String]          // 分类（如 "呼吸", "脉冲", "持续", "高潮"）
}
```

- 用户导入郊狼波形文件后，自动加入预设库
- 打轴时可以从预设库拖拽波形到时间轴
- 同一个波形可在多个时间轴脚本中复用

### 4.4 波形段数据结构

```
WaveformSegment {
  id: String                    // 唯一标识
  channelId: String             // 目标通道ID（对应配置中定义的 channelId）
  label: String                 // 用户标签（如 "第一波高潮"）
  startTimeMs: Long             // 在音频时间轴上的起始时间（毫秒）
  waveformRef: String           // 引用的波形文件ID
  strengthScale: Float          // 强度缩放系数（默认 1.0，如 0.5 = 半强度）
  loopCount: Int                // 波形循环次数，1 = 播放一次，-1 = 循环至 endTimeMs
  endTimeMs: Long               // 结束时间（毫秒），波形不足时循环填充或截断
  disabled: Boolean             // 是否被临时禁用
}
```

### 4.5 时间轴脚本完整结构

```
TimelineScript {
  scriptId: String
  name: String                  // 脚本名称
  configId: String              // 引用的设备配置ID（通道定义来自此配置）

  segments: [WaveformSegment]   // 波形段列表
  disabledChannels: [String]    // 全局禁用的通道ID列表（用于调试）
  totalDurationMs: Long         // 脚本总时长

  metadata: {
    author: String
    description: String
    tags: [String]
    createdAt: Long
    updatedAt: Long
  }
}
```

### 4.6 配置 ID 的一致性保证

- 时间轴脚本创建时必须选择引用的配置 ID
- 配置中定义的通道（channelId）即为脚本中波形段可用的目标通道
- **只有引用同一 configId 的脚本才能加入同一个播放列表**
- 打轴编辑器从配置加载通道列表供用户选择

## 5. 模块三：音频播放引擎

### 5.1 功能描述

**必须自研或有源码**的音频播放器。核心诉求：拖动进度条时能**同步回滚时间轴执行状态**，即 seek 操作必须精确通知到时间轴引擎。

### 5.2 选型约束

- 不可使用系统 `MediaPlayer` 的黑盒 seek 回调（精度不足、行为不可控）
- 采用方案：基于 Android 原生 `AudioTrack` + `MediaCodec` 流式解码自建播放器，`AudioTrack.getPlaybackHeadPosition()` 提供采样级精度位置查询，`MediaExtractor.seekTo()` 实现精确跳转
- 最低要求：能获取**毫秒级精度的当前播放位置和 seek 完成控制**

### 5.3 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| AP-01 | 音频文件加载 | 支持 MP3/WAV 格式 | P0 |
| AP-02 | 播放控制 | 播放、暂停、停止 | P0 |
| AP-03 | 进度跳转 | seek 到指定位置，精确到毫秒 | P0 |
| AP-04 | 播放位置回调 | 持续回调当前播放位置（≥30Hz 频率） | P0 |
| AP-05 | 播放速度控制 | 支持 0.5x ~ 2.0x 变速播放 | P1 |
| AP-06 | 音量控制 | 独立音量调节 | P0 |
| AP-07 | 静音播放 | 音频静音但时间轴继续驱动 | P1 |
| AP-08 | 音频解码 | 解码到 PCM 样本，支持波形渲染 | P0 |
| AP-09 | 多音频实例 | 同时加载多个音频解码器 | P0 |
| AP-10 | 淡入淡出 | 播放/暂停/切换时音频淡入淡出 | P2 |

### 5.4 关键回调接口

```
interface AudioPlayerCallback {
    // 播放位置更新（≥30次/秒）
    fun onPositionUpdate(positionMs: Long)
    
    // seek 开始（拖动进度条时先回调）
    fun onSeekStart(fromPositionMs: Long, toPositionMs: Long)
    
    // seek 完成
    fun onSeekComplete(positionMs: Long)
    
    // 播放状态变化
    fun onPlayStateChange(isPlaying: Boolean)
    
    // 播放完成
    fun onPlaybackComplete()
    
    // 错误
    fun onError(error: AudioPlayerError)
}
```

### 5.5 seek 时的时间轴回滚逻辑

```
用户拖动进度条
1. AudioPlayer 回调 onSeekStart(fromPos, toPos)
2. TimelineEngine 收到后：
   a. 撤销 fromPos → toPos 之间已发送但"不应发送"的指令（如有需要）
   b. 计算 toPos 时刻各通道应有的目标强度
   c. 立即向设备发送强度指令，将设备同步到 toPos 时刻的正确状态
3. AudioPlayer 回调 onSeekComplete(toPos)
4. TimelineEngine 恢复正常的时间轴跟踪模式
```

## 6. 模块四：时间轴执行引擎

### 6.1 功能描述

播放音频时，时间轴执行引擎持续接收播放位置回调，计算当前时刻各通道应输出的强度，并通过蓝牙发送控制指令。它是音频与设备之间的桥梁。

### 6.2 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| TE-01 | 时间轴加载 | 加载并索引时间轴脚本，构建时间查询结构 | P0 |
| TE-02 | 实时强度计算 | 根据当前时间计算所有通道的目标强度 | P0 |
| TE-03 | 指令发送调度 | 将强度变化转化为设备指令并发送 | P0 |
| TE-04 | 多通道并行 | 同时驱动多个设备通道，互不阻塞 | P0 |
| TE-05 | 指令节流 | 避免过于频繁发送蓝牙指令导致拥塞 | P0 |
| TE-06 | seek 回滚 | seek 时快速将设备同步到目标状态 | P0 |
| TE-07 | 暂停恢复 | 暂停时停止指令，恢复时重新同步 | P0 |
| TE-08 | 通道禁用响应 | 运行时响应通道禁用标记 | P0 |
| TE-09 | 播放速度适配 | 变速播放时时间轴速度同步缩放 | P1 |
| TE-10 | 指令队列 | 高优先级指令（安全停止）优先发送 | P1 |
| TE-11 | 性能监控 | 记录指令发送延迟、丢帧情况 | P2 |

### 6.3 强度计算流程

```
每个播放位置更新周期 (~33ms):
1. 获取 currentPositionMs
2. 遍历所有非禁用通道:
   a. 查找覆盖当前时间的所有波形段
   b. 对每个波形段：根据 waveformRef 找到波形文件，按时间偏移取出强度值 × strengthScale
   c. 如果多个段重叠 → 取最高强度（或可配置叠加策略）
3. 对比上次各通道强度值
4. 仅对发生变化的通道生成新指令
5. 将指令加入发送队列
```

### 6.4 多段重叠策略

| 策略 | 描述 |
|------|------|
| `max`（默认） | 取重叠段中最高的强度值 |
| `sum_clamped` | 叠加后钳制到通道最大值 |
| `latest` | 后添加的波形段覆盖先添加的 |

策略在设备配置级别设定，全局生效。

### 6.5 指令节流策略

```
默认节流参数：
- 最小发送间隔：50ms（同一通道连续两次指令的最小间隔）
- 强度变化阈值：≥ 2（强度变化小于此值不发送新指令）
- 这些参数在设备类型级别可配置
```

## 7. 模块五：播放列表系统

### 7.1 功能描述

播放列表是用户实际播放的单元。它将多首音频及其时间轴脚本组织在一起，支持自动匹配同文件夹下的波形/配置文件，也支持手动切换。系统可保存多个播放列表，切换列表时根据配置是否相同决定是否需要重新映射设备。

### 7.2 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| PL-01 | 创建播放列表 | 新建空白列表，绑定时间轴通道配置 | P0 |
| PL-02 | 添加音频 | 导入音频文件（MP3/WAV），自动检测同文件夹下的时间轴和波形文件 | P0 |
| PL-03 | 自动匹配文件 | 添加音频时自动匹配同文件夹下同名/同前缀的 .hvscript 和 .hvwav | P0 |
| PL-04 | 手动切换时间轴 | 每首音频可手动选择/切换关联的时间轴脚本文件 | P0 |

### 7.2.1 自动匹配文件机制

添加音频文件时，系统自动在同文件夹下查找匹配文件：

```
用户选择: /music/session1/hypno_guide.mp3
自动查找:
  /music/session1/hypno_guide.hvscript     → 时间轴脚本
  /music/session1/hypno_guide.hvwav        → 波形文件（如有）

匹配规则（按优先级）：
1. 同文件名、不同后缀（hypno_guide.mp3 → hypno_guide.hvscript）
2. 同文件夹下最新修改的 .hvscript（兜底匹配）
3. 未匹配到则不关联时间轴（纯音频播放）
```

如果用户不满意自动匹配结果，可通过 PL-04 手动切换为其他时间轴文件。

### 7.2.2 三种播放模式

| 模式 | 名称 | 行为 |
|------|------|------|
| `LOOP_LIST` | 列表循环 | 播放完最后一首后回到第一首继续播放，无限循环 |
| `STOP_AT_END` | 列表播完暂停 | 按列表顺序完整播放一轮后停止，不循环 |
| `LOOP_LAST` | 最后一首循环 | 按列表顺序完整播放一轮（第一首→第二首→...→最后一首），最后一首播完后不回到第一首，而是继续循环最后一首 |

播放模式在播放列表级别设置，对整个列表生效。

### 7.2.3 多播放列表与切换

- 系统可保存**多个播放列表**，每个列表独立存储
- 切换播放列表时：
  - 若新旧列表**配置 ID 相同** → 复用已有通道映射，直接进入播放界面
  - 若新旧列表**配置 ID 不同** → 弹出设备关联通道界面，需重新设置映射
- 切换时自动断开不再需要的设备连接，释放资源

### 7.3 通道映射

播放列表打开时**必定显示通道映射界面**。此时用户将已连接的物理设备通道（如"郊狼#1的A通道"）与配置中定义的通道（如"后穴"）进行关联。**物理通道映射在此阶段确定，而非在配置中预设。**

> 注意：配置只定义通道的 ID/名称/设备类型，不绑定物理通道。物理通道的 A/B 映射在通道映射界面由用户决定。

```
通道映射规则：
以设备为主体：
  郊狼#1 A通道 → 选择关联哪个时间轴通道（后穴 / 前部 / 不关联）

1 对 1 映射：
  郊狼#1 A通道 → 时间轴通道"后穴"

多个物理通道可映射到同一时间轴通道：
  郊狼#1 A通道 → 时间轴通道"后穴"
  郊狼#2 A通道 → 时间轴通道"后穴"（多个设备同步震动）

一个物理通道只能映射到一个时间轴通道：
  郊狼#1 A通道 → 只能选一个时间轴通道
```

```

├──────────────────────────────────────────────────────┤
│                                                      │

│                                                      │
│  ┌───────────────────────────────────────────────┐   │

**关键设计要点：**
- **以设备和物理通道为主体**组织界面，而非以时间轴通道为主体
- 每个设备通道旁显示下拉选择器，列出可选的时间轴通道 + "不关联"选项
- **自动恢复上次映射**：同配置 ID 列表打开时，自动填充上次保存的映射
- 设备类型不匹配的时间轴通道自动过滤（郊狼通道只显示 coyote_v3/coyote_v2 类型的时间轴通道）
- 映射结果保存到播放列表中，下次打开同配置列表时自动恢复

### 7.4 全局预加载策略

为避免播放时卡顿，进入播放列表界面时：

```
预加载流程：
1. 读取播放列表所有音频文件的元数据（时长、格式）
2. 为每个音频创建解码器实例（提前打开文件）
3. 加载所有时间轴脚本文件
4. 解析所有波形段数据
5. 为时间轴引擎构建时间查询索引（如 TimelineIndex）
6. 预计算关键时间点的强度值（可选，用于波形预览）
7. 所有数据放入内存缓存

内存管理：
- 解码器实例池：最多同时保持 3 个活跃解码器
- 超出池大小的解码器保存状态但不解码
- 切换曲目时复用解码器实例
```

### 9.6 历史设备与一键重连

- 点击历史设备的"连接"按钮，自动发起 BLE 连接（一键重连）
- 如果设备不在附近（扫描不到），给出超时提示
- 用户可为设备设置别名（如"腰部郊狼"），方便识别

### 9.7 设备协议适配器

每种设备类型需要一个独立的协议适配器，实现统一接口：

```
interface DeviceProtocolAdapter {
    // 设备类型标识
    val deviceType: String
    
    // 将通道强度值编码为蓝牙指令字节
    fun encodeCommand(channelId: String, strength: Int): ByteArray
    
    // 强度范围校验
    fun validateStrength(channelId: String, strength: Int): Boolean
    
    // 停止指令（强度归零）
    fun encodeStopCommand(): ByteArray
    
    // BLE 服务/特征值 UUID
    fun getServiceUUID(): UUID
    fun getCharacteristicUUID(): UUID
}
```

**郊狼 (Coyote DG-LAB) 协议要点：**
- A 通道和 B 通道独立控制
- 每个通道强度范围 0-200
- 需要持续发送心跳/保活指令维持连接
- 指令格式参考 DG-LAB 官方通信协议

### 9.8 设备类型注册

object DeviceTypeRegistry {
    // 注册所有支持的设备类型
    private val adapters: Map<String, DeviceProtocolAdapter> = mapOf(
        "coyote_v3" to CoyoteV3Adapter(),

        "lovense_vibrate" to LovenseVibrateAdapter(),
        "generic_pwm" to GenericPwmAdapter()
    )
    
    // 获取指定类型的适配器

    fun getSupportedTypes(): List<DeviceTypeInfo>
}
```

### 9.9 自动重连机制

- 连接意外断开时自动尝试重连，最多 3 次静默重连
- 每次重连间隔递增（1s / 2s / 4s）
- 3 次均失败后通知用户并停止重连
- 用户可手动触发重连

### 10.2 功能清单

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| WM-01 | 波形导入 | 导入 DG-LAB 及其他格式的波形文件 | P0 |
| WM-02 | 波形格式转换 | 将不同来源格式统一转换为 .hvwav | P0 |
| WM-03 | 波形浏览 | 列表/网格查看所有已导入波形 | P0 |
| WM-04 | 波形预览 | 显示波形强度曲线图 | P1 |
| WM-05 | 波形分类 | 按标签/类型对波形分组 | P1 |
| WM-06 | 波形搜索 | 按名称/标签搜索波形 | P1 |
| WM-07 | 波形删除 | 删除不再使用的波形文件 | P1 |
| WM-08 | 波形标签编辑 | 为波形添加/修改标签 | P1 |
| WM-09 | 实时强度测试 | 连接设备后实时测试波形效果 | P0 |
| WM-10 | 强度滑块控制 | 分别控制各通道强度 0-200 | P0 |
| WM-11 | 强度预设管理 | 保存/加载常用强度组合 | P0 |
| WM-12 | 安全停止 | 紧急停止按钮（强度归零） | P0 |

## 11. 数据流与生命周期

### 11.1 完整播放数据流

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 音频文件  │───▶│ 音频播放  │───▶│ 音频输出  │
│(MP3/WAV) │    │引擎      │    │(扬声器)   │
└──────────┘    └─────┬────┘    └──────────┘
                      │ 播放位置回调
                      ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│时间轴脚本│───▶│ 时间轴执行│───▶│ 设备协议  │───▶│ 蓝牙发送  │
│(.json)   │    │引擎      │    │适配器    │    │(BLE)     │
└──────────┘    └─────┬────┘    └─────┬────┘    └─────┬────┘
                      │               │               │
                      ▼               ▼               │
                ┌──────────┐         │               │
                │ 通道映射  │         │               │
                │ 表        │         │               │
                └──────────┘         │               │
                                     ▼               │
                               ┌──────────┐          │
                               │ 物理设备  │◀─────────┘
                               │(郊狼等)  │
                               └──────────┘
```

### 11.2 播放生命周期

```
1. 启动 APP
2. 蓝牙扫描 & 连接设备
3. 创建/选择设备配置
4. 创建/选择播放列表
5. 打开播放列表 → 触发全局预加载
6. 设置通道映射（如未设置）
7. 用户点击播放
8. 音频播放引擎启动 + 时间轴执行引擎启动
9. 播放中：持续位置回调 → 强度计算 → 指令发送
10. 用户 seek → 引擎回滚 → 重新同步
11. 播放结束/用户停止
12. 安全停止：所有设备强度归零
```

### 11.3 打轴编辑器生命周期

```
1. 选择设备配置
2. 加载音频文件 → 渲染波形
3. 创建/打开时间轴脚本
4. 编辑波形段（添加、调整、标签）
5. [可选] 连接设备实时预览
6. 保存时间轴脚本文件
```

**channelMapping 说明：**
- key：`设备ID:物理通道`（如 `coyote_1:A`），**以设备为主体**
- `timelineChannelId`：关联的时间轴通道ID，`null` 表示不关联
- `deviceAlias` 和 `macAddress`：用于下次打开时识别设备并自动填充
- 多个物理通道可指向同一个时间轴通道（如示例中两个郊狼的 A 通道都指向"后穴"）

### 12.6 兼容性策略

- `version` 字段用于未来格式升级
- 读取时检查版本号，做向前兼容
- 未知字段忽略（不报错）
- 缺少必要字段时给出明确错误提示

## 附录 A：术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 设备配置 | DeviceConfig | 拥有唯一ID，定义各通道名称/设备/强度范围 |
| 配置ID | ConfigId | 设备配置的唯一标识，时间轴脚本通过此ID引用配置 |
| 通道 | Channel | 配置中定义的控制通道，含ID/名称/设备类型/强度范围，不绑定物理通道 |
| 时间轴脚本 | TimelineScript | 引用配置ID + 波形段数据 |
| 波形段 | WaveformSegment | 时间轴上引用一段波形文件的数据 |
| 波形文件 | WaveformFile | 导入的强度-时间采样数据，波形的基本单位 |
| 波形预设库 | WaveformPresetLibrary | 已导入波形文件的集合，打轴时可复用 |
| 通道映射 | ChannelMapping | 配置通道 → 物理设备通道的绑定（以设备为主体） |
| 播放列表 | Playlist | 音频+时间轴的有序集合，含播放模式 |
| 打轴 | TimelineEditing | 可视化制作时间轴脚本 |
| 播放模式 | PlayMode | 列表循环 / 列表播完暂停 / 最后一首循环 |
| 历史设备 | PairedDevice | 曾经连接过、已记录的蓝牙设备 |

## 附录 C：扩展性设计原则

1. **设备类型可插拔**：新增设备类型只需实现 `DeviceProtocolAdapter` 接口和注册类型定义
2. **波形格式可扩展**：`WaveformFile.sourceFormat` 字段标识来源，新增转换器即可支持新格式
3. **配置兼容**：`version` 字段 + 忽略未知字段策略保证向前兼容
4. **播放引擎可替换**：通过 `AudioPlayer` 接口抽象，后续换底层实现不影响上层
