## Phase 3：音频引擎 (纯 Java)

**目标**：实现基于 AudioTrack + MediaCodec 的流式音频播放器，支持任意音频格式和毫秒级位置查询。

### 3.1 Java 音频引擎开发

| 任务 | 文件 | 说明 |
|------|------|------|
| AudioTrack 播放引擎 | `AudioEngine.java` | 流式解码播放、play/pause/seek/getPosition/getDuration |
| MediaCodec 流式解码 | `AudioEngine.java` 内 | 配合 MediaExtractor 边读边解，内存占用低 |
| 位置轮询 | `PlaySessionVM.java` 内 | 50ms 定时器调用 AudioTrack.getPlaybackHeadPosition() |
| 精确 seek | `AudioEngine.java` 内 | extractor.seekTo() + flush codec + flush audioTrack |

### 3.2 播放集成

| 任务 | 文件 | 说明 |
|------|------|------|
| AudioEngine | `AudioEngine.java` | 纯 Java，无 native 方法 |
| ViewModel 接入 | `PlaySessionVM.java` | loadFile → play → positionPoller |

### 3.3 播放界面

| 任务 | 说明 |
|------|------|
| `PlaylistScreen` 播放控制 | 播放/暂停/停止按钮 |
| 进度条 | 可拖动 seek，实时显示当前时间 |
| 曲目切换 | 上一首/下一首 |

### 3.4 验证标准

- 导入 MP3 → 点击播放 → 听到声音
- 拖动进度条 → 音频跳到新位置继续播放
- 导入 WAV → 播放正常
- 播放到末尾 → `onPlaybackComplete` 回调触发

## Phase 5：郊狼 V3 协议适配

**目标**：完整实现郊狼协议子包，包含 B0/BF/B1 全部协议逻辑。

> **注意**：郊狼协议作为完整自治子包封装在 `infrastructure/ble/adapter/coyote/` 下，内部实现详见 [郊狼协议移植方案.md](郊狼协议移植方案.md)。本阶段聚焦于子包内部协议逻辑的完整实现和联调。

### 5.1 郊狼协议子包实现

| 任务 | 说明 |
|------|------|
| 协议编解码层 | V3 的 B0/BF/B1 指令编解码，V2 的 PWM_AB2/A34/B34 编解码（package-private） |
| 频率换算 | 10-1000ms → 10-240 协议值的压缩转换（package-private） |
| 4×25ms 槽位拆分 | 从波形帧数据拆分为 4 组 `{frequency, strength}` |
| CoyoteV3Adapter | 组合协议编解码 + BLE GATT + 100ms 定时器 + 序列号流控 |
| BF 指令 | 连接时写入默认 BF（软上限 200，平衡 128） |
| B1 Notify 处理 | 序列号匹配确认，强度变化通知 |
| 序列号流控状态机 | isInputAllowed + accumulatedDelta 模式 |
| 单通道输出 | 另一通道强度 >100 放弃该通道 |

### 5.2 波形测试

| 任务 | 说明 |
|------|------|
| `CoyoteTestScreen` 波形模式 | 新增：选择预设波形（恒定/呼吸/脉冲简化版）→ 发送 B0 |
| 硬编码测试波形 | 3 组内置波形数据（呼吸感、脉冲感、恒定低强度） |
| 强度+波形联动 | 同时调节通道强度 + 发送波形数据 |

### 5.3 验证标准

- 连接郊狼 V3 → 选择"呼吸感波形" → 郊狼输出呼吸般起伏的体感
- 波形测试中调节强度 → B1 返回确认 → 体感变化
- 连续运行 1 分钟 → 无丢帧无异常

## Phase 6：时间轴引擎 + 播放联动

**目标**：实现完整播放链路——音频播放 + 时间轴索引 + 设备驱动联动。这是核心 MVP。

### 6.1 Protobuf 序列化

| 任务 | 说明 |
|------|------|
| 编译 proto 文件 | `timeline_segment.proto`，`coyote_waveform.proto`，`lovense_waveform.proto` |
| ProtoSerializer | Protobuf → Base64 → JSON 转换 |

### 6.2 TimelineEngine

| 任务 | 文件 | 说明 |
|------|------|------|
| TimelineIndex (Java) | `timeline_index` | 排序数组二分查找 `(startMs, endMs, segmentId)` |
| TimelineEngine (Java wrapper) | `TimelineEngine.java` | `buildIndex(segments[])` → `query(positionMs)` |
| TimelineSnapshot | 数据类 | `Map<channelId， { waveformData(bytes)， offsetMs }>` |

### 6.3 PlaybackCoordinator

| 任务 | 说明 |
|------|------|
| `PlaybackCoordinator` | 线程安全版：`ConcurrentHashMap + CopyOnWriteArrayList` |
| `onTimelineTick(positionMs， snapshot)` | Audio 线程回调，推送快照到所有 Adapter |
| `addAdapter/removeAdapter` | BLE 线程调用 |
| seek 联动 | `onSeek()` → flush 所有 Adapter → 新位置快照 |

### 6.4 播放列表联动

| 任务 | 说明 |
|------|------|
| 预加载 | 打开播放列表时：加载 JSON → 校验 → 构建 TimelineIndex |
| 通道映射集成 | `ChannelMappingScreen` → 物理设备通道 ↔ 配置通道 |
| 时间轴脚本关联 | 音频 ↔ `.hvscript`（自动匹配 + 手动选择） |
| 播放链路 | 播放按钮 → AudioEngine 启动 → TimelineEngine 查询 → PlaybackCoordinator → Adapter → BLE |

### 6.5 播放模式

| 任务 | 说明 |
|------|------|
| `LOOP_LIST` | 列表循环 |
| `STOP_AT_END` | 播完暂停 |
| `LOOP_LAST` | 最后一首单曲循环 |

### 6.6 验证标准

- 创建时间轴脚本（手动编辑 JSON 或使用简单的 DSL）→ 关联音频
- 创建播放列表 → 匹配配置 ID → 添加曲目 → 通道映射
- 点击播放 → 音频播放，郊狼同步输出波形效果
- 拖动进度条 → seek → 郊狼输出立即同步到新位置
- 暂停 → 郊狼归零 → 恢复 → 输出恢复

## Phase 8：打轴编辑器

**目标**：实现可视化时间轴编辑器，用户可从预设库拖拽波形段到轨道。

### 8.1 编辑器核心

| 任务 | 说明 |
|------|------|
| `WaveformCanvas` | Compose Canvas 自定义绘制：音频波形 + 多通道轨道 + 时间标尺 |
| 播放头 | 红色竖线，随音频播放移动 |
| 波形段渲染 | 每个段显示名称 + 波形 mini 预览 + 强度缩放系数 |
| 时间缩放 | 水平缩放按钮（放大/缩小时间轴） |
| 视口裁剪 | 只渲染可见范围的波形段，长音频不卡顿 |

### 8.2 交互

| 任务 | 说明 |
|------|------|
| 拖拽创建 | 从预设库拖拽波形到轨道 → 创建 WaveformSegment |
| 拖动边缘 | 调整起止时间，自动吸附到网格/其他段边缘 |
| 拖动中部 | 移动段位置 |
| 双击段 | 打开属性面板（缩放系数、循环次数、标签） |
| 右键/长按段 | 上下文菜单（复制/删除/禁用/替换波形） |
| 通道禁用 | 轨道右侧开关，临时静默某通道 |

### 8.3 编辑器功能

| 任务 | 说明 |
|------|------|
| 撤销/重做 | 完整操作历史栈 |
| 复制粘贴 | 跨通道/跨时间复制波形段 |
| 播放试听 | 编辑器内播放音频，播放头移动（不发送 BLE） |
| 实时预览(可选) | 连接设备时，编辑同时发送 B0 |
| 保存/导出 | 写入 `.hvscript` 文件 |

### 8.4 验证标准

- 加载音频 → 显示波形图 + 时间轴轨道（轨道来自配置的通道）
- 从预设库拖拽"呼吸感"波形到"后穴"轨道 → 创建段
- 拖动段边缘调整时长 → 双击改标签为"第一波高潮"
- Ctrl+Z → 标签回到"呼吸感-慢" → Ctrl+Shift+Z 恢复
- 保存 → 关闭 → 重新打开编辑器 → 内容完整
- 导出脚本 → 在播放列表关联 → 播放生效

## Phase 9：后台播放 + 完善

**目标**：实现完整产品体验——后台播放、V2 兼容、错误恢复、性能调优。

### 9.1 后台播放

| 任务 | 说明 |
|------|------|
| `PlaybackService` | 前台 Service，`startForeground()` 带通知栏 |
| 通知栏控制 | 播放/暂停/上下首 + 当前曲目 + 实时强度 |
| AudioEngine 迁移 | 从 Activity 移到 Service 持有 |
| Activity-Service 绑定 | `bindService()` 获取状态和控制器 |
| 生命周期 | 后台/前台切换不中断播放 |
| `onDestroy` 安全停止 | 强制归零 |

### 9.2 完善

| 任务 | 说明 |
|------|------|
| CoyoteV2Adapter | V2 UUID + PWM_AB2/A34/B34 + 大小端转换 |
| V2/V3 自动识别 | 根据蓝牙名称区分并加载对应 Adapter |
| 断连重连 | 3 次静默重试（1s/3s/5s），失败提示用户 |
| 错误处理完善 | 音频解码失败跳过、BLE 写入失败重试、B1 超时警告 |
| 通知渠道 | Android 8+ 创建通知渠道 |

### 9.3 验证标准

- 播放中按 Home 键 → 通知栏显示播放状态 → 音频和设备持续工作
- 锁屏 → 通知栏可暂停/切歌
- 连接 V2 郊狼 → BLE 名称识别为 V2 → 加载 V2 Adapter → 正常输出
- 播放中关闭郊狼电源 → 静默重连 3 次 → 失败后弹出"设备已断开"提示 → 其他设备/音频不受影响
- Service.onDestroy → 所有设备强度归零
