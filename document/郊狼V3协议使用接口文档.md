## 1. 连接初始化流程

```
连接成功后的初始化序列（严格顺序）：

  1. 绑定 Notify
     gatt.setCharacteristicNotification(0x150B, true)
     
  2. 写入 BF 指令（设置软上限）
     gatt.writeCharacteristic(0x150A, buildBFCommand())
     
  3. 启动 100ms 定时器
     开始周期性发送 B0 指令
```

> BF 指令必须在 B0 循环之前写入，否则软上限可能是未知的历史值。

## 2. B0 指令：核心控制指令

### 2.1 字节结构（20 bytes）

```
字节0:     0xB0 (指令头)
字节1:     高4bit=序列号 | 低4bit=强度解读方式
字节2:     A通道强度设定值 (0-200)
字节3:     B通道强度设定值 (0-200)
字节4-7:   A通道波形频率 ×4 (每byte=25ms, 有效值10-240)
字节8-11:  A通道波形强度 ×4 (每byte=25ms, 有效值0-100)
字节12-15: B通道波形频率 ×4
字节16-19: B通道波形强度 ×4
```

### 2.2 强度解读方式（字节1 低4bit）

```
bit3-2 (A通道)  bit1-0 (B通道)
    00              00       = 强度不变（设定值无效）
    01              01       = 相对增加（强度 += 设定值）
    10              10       = 相对减少（强度 -= 设定值）
    11              11       = 绝对设置（强度 = 设定值）
```

**我们的场景下，时间轴引擎始终使用绝对设置（0b11）**，因为引擎每帧计算的是目标强度值。只有当用户手动拨动滚轮/按+/-按钮时才使用相对增减。

### 2.3 序列号（字节1 高4bit）

| 值 | 含义 | 使用场景 |
|:--:|------|------|
| 0 | 不要求反馈 | 纯波形输出（仅频率+强度变化，通道强度不变） |
| 1-15 | 要求设备反馈 B1 | 修改了通道强度，需要确认结果 |

**关键约束**：序列号 ≠ 0 时，必须等待设备返回相同序列号的 B1 消息后，才允许再次发送序列号 ≠ 0 的指令。这是协议层面的流控机制。

### 2.4 单通道输出技巧

只想让 A 通道输出波形时，将 B 通道的某个强度值设为 >100（如 101）：

```
A通道: 频率{10,10,10,10}  强度{0,10,20,30}
B通道: 频率{0,0,0,0}      强度{0,0,0,101}    ← 被设备放弃
```

### 2.5 构造伪代码

```java
byte[] buildB0Command(
    boolean changeStrength,      // 是否修改通道强度
    int seqNo,                   // 序列号 0-15
    int strengthModeA,           // A通道解读方式 0-3
    int strengthModeB,           // B通道解读方式 0-3
    int strengthA,               // A通道强度设定值 0-200
    int strengthB,               // B通道强度设定值 0-200
    int[] freqA,                 // A通道频率×4 (10-240)
    int[] strengthWaveA,         // A通道波形强度×4 (0-100)
    int[] freqB,                 // B通道频率×4 (10-240)
    int[] strengthWaveB          // B通道波形强度×4 (0-100)
) {
    byte[] cmd = new byte[20];
    cmd[0] = (byte) 0xB0;
    cmd[1] = (byte) ((seqNo << 4) | ((strengthModeA << 2) | strengthModeB));
    cmd[2] = (byte) strengthA;
    cmd[3] = (byte) strengthB;
    
    for (int i = 0; i < 4; i++) cmd[4 + i]  = (byte) freqA[i];
    for (int i = 0; i < 4; i++) cmd[8 + i]  = (byte) strengthWaveA[i];
    for (int i = 0; i < 4; i++) cmd[12 + i] = (byte) freqB[i];
    for (int i = 0; i < 4; i++) cmd[16 + i] = (byte) strengthWaveB[i];
    
    return cmd;
}
```

## 3. BF 指令：软上限与平衡参数

### 3.1 字节结构（7 bytes）

```
字节0:     0xBF (指令头)
字节1:     A通道强度软上限 (0-200, 超出不变)
字节2:     B通道强度软上限 (0-200, 超出不变)
字节3:     A通道频率平衡参数1 (0-255)
字节4:     B通道频率平衡参数1 (0-255)
字节5:     A通道频率平衡参数2 (0-255)
字节6:     B通道频率平衡参数2 (0-255)
```

### 3.2 参数说明

| 参数 | 作用 | 推荐默认值 |
|------|------|:---:|
| 软上限 | 通道强度上限限制，断电保存 | 200（不限制） |
| 平衡参数1 | 调整低频波形冲击感，值越大低频越强 | 128（中性） |
| 平衡参数2 | 调整波形脉冲宽度，值越大低频刺激越强 | 128（中性） |

### 3.3 构造伪代码

```java
byte[] buildBFCommand(int softLimitA, int softLimitB,
                       int balance1A, int balance1B,
                       int balance2A, int balance2B) {
    return new byte[] {
        (byte) 0xBF,
        (byte) softLimitA,
        (byte) softLimitB,
        (byte) balance1A,
        (byte) balance1B,
        (byte) balance2A,
        (byte) balance2B
    };
}

// 默认配置（不限制软上限，中性平衡）
byte[] defaultBF = buildBFCommand(200, 200, 128, 128, 128, 128);
```

### 3.4 重要规则

- 写入即生效，**无返回值**
- 断电保存
- **每次 BLE 重连后必须重新写入**（设备可能被其他 APP 改过）
- 执行顺序：先 BF → 再启动 B0 循环

## 4. B1 消息：强度反馈

### 4.1 消息结构（4 bytes，来自 0x150B Notify）

```
字节0:     0xB1 (消息头)
字节1:     序列号 (与触发此反馈的 B0 指令序列号一致/0)
字节2:     A通道当前实际强度 (0-200)
字节3:     B通道当前实际强度 (0-200)
```

### 4.2 触发条件

- B0 指令中序列号 ≠ 0 且修改了通道强度 → B1 返回相同序列号
- 物理滚轮拨动 → B1 返回序列号=0
- 任何其他原因导致强度变化 → B1 返回序列号=0

### 4.3 处理伪代码

```java
void onB1Received(byte[] data) {
    int seqNo = data[1] & 0xFF;
    int strengthA = data[2] & 0xFF;
    int strengthB = data[3] & 0xFF;
    
    // 更新本地记录的设备强度
    this.deviceStrengthA = strengthA;
    this.deviceStrengthB = strengthB;
    
    // 如果是对我们发出的强度修改指令的确认
    if (seqNo == this.pendingSeqNo) {
        this.strengthChangeConfirmed = true;
        this.pendingSeqNo = 0;
        // 允许下一次强度修改
    }
    
    // 通知上层（UI 更新等）
    listener.onStrengthChanged(strengthA, strengthB, seqNo == 0);
}
```

## 5. 波形频率换算

### 5.1 换算公式

V3 协议的波形频率经过压缩，用户输入值（10-1000ms）需转换为协议值（10-240）：

```java
int convertFrequency(int inputMs) {
    if (inputMs >= 10 && inputMs <= 100) {
        return inputMs;
    } else if (inputMs <= 600) {
        return (inputMs - 100) / 5 + 100;
    } else if (inputMs <= 1000) {
        return (inputMs - 600) / 10 + 200;
    }
    return 10; // fallback
}
```

### 5.2 常用速查

| 体感效果 | 脉冲频率 | 波形频率(ms) | 协议值 |
|------|:---:|:---:|:---:|
| 极高频 | 100Hz | 10 | 10 |
| 高频 | 50Hz | 20 | 20 |
| 中高频 | 20Hz | 50 | 50 |
| 中频 | 10Hz | 100 | 100 |
| 低频 | 5Hz | 200 | 120 |
| 极低频 | 1Hz | 1000 | 240 |

## 6. 典型使用场景

### 场景一：纯波形播放（不修改通道强度）

时间轴引擎根据波形文件，每 100ms 构造新的频率+强度数据：

```java
// 每100ms调用一次
void sendWaveformOnly(int[] freqA, int[] waveStrengthA,
                      int[] freqB, int[] waveStrengthB) {
    byte[] cmd = buildB0Command(
        false,          // 不修改通道强度
        0,              // 序列号=0（不要求反馈）
        0, 0,           // 解读方式=不变
        0, 0,           // 强度设定值无效
        freqA, waveStrengthA,
        freqB, waveStrengthB
    );
    gatt.writeCharacteristic(writeChar, cmd);
}
```

### 场景二：安全停止（强度归零）

```java
void emergencyStop() {
    byte[] cmd = buildB0Command(
        true,           // 修改通道强度
        1,              // 序列号=1，要求确认
        3, 3,           // 两通道均为绝对设置(0b11)
        0, 0,           // 强度=0
        new int[]{10,10,10,10}, new int[]{0,0,0,0},  // A通道静默
        new int[]{10,10,10,10}, new int[]{0,0,0,0}   // B通道静默
    );
    gatt.writeCharacteristic(writeChar, cmd);
}
```

> 如果想要更激进地归零，可以将解读方式设为 0b1100 + 强度=0，设备直接设为 0。

### 场景三：用户手动调节强度并等待确认

```java
class StrengthController {
    int pendingSeqNo = 0;
    int accumulatedDeltaA = 0;
    int accumulatedDeltaB = 0;
    boolean waitingConfirm = false;
    
    // 用户按了 A通道 +1
    void onUserIncreaseA() {
        accumulatedDeltaA += 1;
    }
    
    // 100ms 定时器回调
    void onTimerTick() {
        if (waitingConfirm) {
            // 还在等上次确认，强度部分填0（不变）
            sendB0StrengthUnchanged();
            return;
        }
        
        if (accumulatedDeltaA != 0 || accumulatedDeltaB != 0) {
            int modeA = accumulatedDeltaA > 0 ? 1 : (accumulatedDeltaA < 0 ? 2 : 0);
            int modeB = accumulatedDeltaB > 0 ? 1 : (accumulatedDeltaB < 0 ? 2 : 0);
            
            pendingSeqNo = (pendingSeqNo % 15) + 1;
            waitingConfirm = true;
            
            byte[] cmd = buildB0Command(
                true, pendingSeqNo,
                modeA, modeB,
                Math.abs(accumulatedDeltaA), Math.abs(accumulatedDeltaB),
                currentFreqA, currentWaveA, currentFreqB, currentWaveB
            );
            gatt.writeCharacteristic(writeChar, cmd);
            
            accumulatedDeltaA = 0;
            accumulatedDeltaB = 0;
        } else {
            sendB0StrengthUnchanged();
        }
    }
    
    // B1 回调
    void onB1Received(int seqNo, int strengthA, int strengthB) {
        if (seqNo == pendingSeqNo) {
            waitingConfirm = false;
            pendingSeqNo = 0;
        }
        deviceStrengthA = strengthA;
        deviceStrengthB = strengthB;
    }
}
```

### 场景四：时间轴引擎驱动（完整周期）

```java
// 每100ms调用
void onTimelineTick(long currentTimeMs) {
    // 1. 从时间轴引擎获取当前时刻各通道目标值
    TimelineSnapshot snap = engine.calculate(currentTimeMs);
    
    // 2. 转换为协议值
    int[] freqA = new int[4];
    int[] waveA = new int[4];
    // 将本100ms窗口内4个25ms槽位的数据填入
    // 注意：需要频率+强度两个维度
    fillSlots(snap.channelA, freqA, waveA);
    
    int[] freqB = new int[4];
    int[] waveB = new int[4];
    fillSlots(snap.channelB, freqB, waveB);
    
    // 3. 确定强度修改模式
    boolean changeStrength = needStrengthUpdate(snap);
    int mode = changeStrength ? 3 : 0; // 绝对设置 or 不变
    int seqNo = changeStrength ? nextSeqNo() : 0;
    
    // 4. 构造并发送
    byte[] cmd = buildB0Command(
        changeStrength, seqNo,
        mode, mode,
        snap.strengthA, snap.strengthB,
        freqA, waveA, freqB, waveB
    );
    gatt.writeCharacteristic(writeChar, cmd);
    
    // 5. 如果修改了强度，标记等待确认
    if (changeStrength && seqNo != 0) {
        waitingConfirm = true;
        pendingSeqNo = seqNo;
    }
}
```

## 7. 完整 API 封装建议

### 7.1 CoyoteDevice 类

```java
public class CoyoteDevice {
    private BluetoothGatt gatt;
    private BluetoothGattCharacteristic writeChar;   // 0x150A
    private BluetoothGattCharacteristic notifyChar;  // 0x150B
    
    // 设备状态
    private int strengthA = 0;
    private int strengthB = 0;
    private int softLimitA = 200;
    private int softLimitB = 200;
    private int batteryLevel = -1;
    
    // 流控状态
    private int pendingSeqNo = 0;
    private boolean waitingConfirm = false;
    
    // === 生命周期 ===
    
    /** 连接成功后调用 */
    public void onConnected(BluetoothGatt gatt) {
        this.gatt = gatt;
        this.writeChar = gatt.getService(UUID_180C).getCharacteristic(UUID_150A);
        this.notifyChar = gatt.getService(UUID_180C).getCharacteristic(UUID_150B);
        
        // 绑定 Notify
        gatt.setCharacteristicNotification(notifyChar, true);
        
        // 写入 BF 默认值
        writeBF(softLimitA, softLimitB, 128, 128, 128, 128);
        
        // 启动 100ms 定时器
        startLoop();
    }
    
    // === 指令写入 ===
    
    /** 写入 BF 软上限配置 */
    public void writeBF(int limitA, int limitB,
                        int balance1A, int balance1B,
                        int balance2A, int balance2B);
    
    /** 写入 B0（由 100ms 定时器调用） */
    public void writeB0(boolean changeStrength, int seqNo,
                        int modeA, int modeB,
                        int strA, int strB,
                        int[] freqA, int[] waveA,
                        int[] freqB, int[] waveB);
    
    /** 安全停止 */
    public void emergencyStop();
    
    // === 波形辅助 ===
    
    /** 将用户频率转换为协议频率 */
    public static int convertFrequency(int userFreqMs);
    
    /** 将波形段数据拆分为 4 个 25ms 槽位 */
    public static void splitToSlots(WaveformData data, long offsetMs,
                                     int[] outFreq, int[] outStrength);
    
    // === 回调处理 ===
    
    /** 处理 0x150B Notify 数据 */
    public void onNotifyReceived(byte[] data);
    
    // === 监听器 ===
    
    public interface Listener {
        void onStrengthChanged(int a, int b, boolean fromDevice);
        void onBatteryLevel(int level);
        void onDisconnected();
    }
}
```

## 8. 电量读取

### 8.1 特性

- 服务 UUID: `0x180A`
- 特性 UUID: `0x1500`
- 属性: 读/通知
- 数据: 1 byte (0-100，表示电量百分比)

### 8.2 读取方式

```java
// 被动接收（推荐）
gatt.setCharacteristicNotification(batteryChar, true);
// 当电量变化时自动通过 onCharacteristicChanged 回调

// 主动读取
gatt.readCharacteristic(batteryChar);
// 结果通过 onCharacteristicRead 回调
```
