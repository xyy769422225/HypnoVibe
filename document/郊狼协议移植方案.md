## 1. 移植策略

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **完整自治子包** | 郊狼协议全部代码封装在 `coyote/` 子包下，内部自主决定类划分、数据模型、状态机 |
| **最小暴露面** | 对外只暴露 2 个 public 类（`CoyoteV3Adapter` / `CoyoteV2Adapter`），其余全部 package-private |
| **零改动现有代码** | 不修改任何已有 `.java` / `.kt` 文件，全部通过新增文件实现 |
| **零新增依赖** | BLE 使用 Android SDK 内置 `android.bluetooth.*`，协议编解码纯手写 |
| **保证编译通过** | 本期目标仅保证 `compileDebugJavaWithJavac` 通过，不在代码中实际调用 |

### 1.2 与架构文档的关系

架构设计文档定义了统一的 `DeviceProtocolAdapter` 接口（对外契约），**不约束子包内部实现**。本方案定义 `coyote/` 子包的内部设计，是架构文档的补充而非覆盖。

### 1.3 协议来源说明

`dglab-bluetooth-protocol` 是纯 Markdown 文档仓库，不含任何源代码。因此"移植"的实际含义是：**参照协议文档规范，在 HypnoVibe 中用 Java 实现完整的协议代码**。协议规范详见：

- [郊狼蓝牙协议认知文档.md](郊狼蓝牙协议认知文档.md) — V2/V3 协议完整详解
- [郊狼V3协议使用接口文档.md](郊狼V3协议使用接口文档.md) — V3 接口设计与使用场景

### 4.2 CoyoteFrequencyConverter — 频率换算

V3 协议的波形频率经过压缩转换。用户输入值（10-1000ms）需转换为协议值（10-240）。

```java
package com.hypno.hypnovibe.infrastructure.ble.adapter.coyote;

// package-private
final class CoyoteFrequencyConverter {
    private CoyoteFrequencyConverter() {}

    /**
     * 将用户输入的波形频率（ms）转换为 V3 协议值（10-240）。
     *
     * 换算规则：
     *   10..100   → 直接使用输入值
     *   101..600  → (输入值-100)/5 + 100
     *   601..1000 → (输入值-600)/10 + 200
     *   其他      → 返回 10（最小值）
     */
    static int convert(int userFreqMs) {
        if (userFreqMs >= 10 && userFreqMs <= 100) {
            return userFreqMs;
        } else if (userFreqMs <= 600) {
            return (userFreqMs - 100) / 5 + 100;
        } else if (userFreqMs <= 1000) {
            return (userFreqMs - 600) / 10 + 200;
        }
        return CoyoteConstants.WAVE_FREQ_MIN;
    }

    /**
     * 逆换算：将协议值（10-240）还原为近似用户值（ms）。
     * 用于从协议数据反向解析。
     */
    static int inverse(int protocolValue) {
        if (protocolValue >= 10 && protocolValue <= 100) {
            return protocolValue;
        } else if (protocolValue <= 200) {
            return (protocolValue - 100) * 5 + 100;
        } else if (protocolValue <= 240) {
            return (protocolValue - 200) * 10 + 600;
        }
        return 10;
    }
}
```

## 5. 协议编解码规范

### 5.1 CoyoteV3Protocol — V3 协议编解码（package-private）

纯静态方法，零状态，负责 V3 所有指令的字节级编解码。

```java
package com.hypno.hypnovibe.infrastructure.ble.adapter.coyote;

// package-private
final class CoyoteV3Protocol {
    private CoyoteV3Protocol() {}

    // ===== B0 指令构造（20 bytes） =====
    static byte[] buildB0(
        int seqNo,              // 序列号 0-15, 0=不要求反馈
        int strengthModeA,      // A通道强度解读方式 0-3
        int strengthModeB,      // B通道强度解读方式 0-3
        int strengthA,          // A通道强度设定值 0-200
        int strengthB,          // B通道强度设定值 0-200
        int[] freqA,            // A通道波形频率×4（已换算的协议值 10-240）
        int[] strengthWaveA,    // A通道波形强度×4 (0-100)
        int[] freqB,            // B通道波形频率×4
        int[] strengthWaveB     // B通道波形强度×4
    );

    // ===== BF 指令构造（7 bytes） =====
    static byte[] buildBF(
        int softLimitA, int softLimitB,           // 软上限 0-200
        int balance1A, int balance1B,             // 频率平衡参数1 0-255
        int balance2A, int balance2B              // 频率平衡参数2 0-255
    );

    // ===== B1 消息解析（4 bytes） =====
    static CoyoteB1Message parseB1(byte[] data);

    // ===== 安全停止指令 =====
    // 绝对设置模式(0b11) + 强度0，要求序列号确认
    static byte[] buildEmergencyStop();

    // ===== 100ms 窗口拆分为 4 个 25ms 槽位 =====
    static void splitToSlots(CoyoteWaveFrame[] frames, long offsetMs,
                             int[] outFreq, int[] outStrength);
}
```

**B0 字节布局（20 bytes）：**

```
字节0:     0xB0 (指令头)
字节1:     高4bit=序列号 | 低4bit=强度解读方式(bit3-2=A, bit1-0=B)
字节2:     A通道强度设定值 (0-200)
字节3:     B通道强度设定值 (0-200)
字节4-7:   A通道波形频率 ×4 (每byte=25ms, 有效值10-240)
字节8-11:  A通道波形强度 ×4 (每byte=25ms, 有效值0-100)
字节12-15: B通道波形频率 ×4
字节16-19: B通道波形强度 ×4
```

### 5.2 CoyoteV2Protocol — V2 协议编解码（package-private）

```java
package com.hypno.hypnovibe.infrastructure.ble.adapter.coyote;

// package-private
final class CoyoteV2Protocol {
    private CoyoteV2Protocol() {}

    // ===== PWM_AB2 构造（3 bytes, 通道强度） =====
    // bits 23-22: 保留
    // bits 21-11: A通道强度 (0-2047)
    // bits 10-0:  B通道强度 (0-2047)
    static byte[] buildPwmAB2(int strengthA, int strengthB);

    // ===== PWM_A34 构造（3 bytes, A通道波形，注意：写入B通道数据） =====
    // bits 23-20: 保留
    // bits 19-15: Z值 (5 bits, 0-31) 脉冲宽度
    // bits 14-5:  Y值 (10 bits, 0-1023) 脉冲间隔
    // bits 4-0:   X值 (5 bits, 0-31) 连续脉冲数
    static byte[] buildPwmA34(int x, int y, int z);

    // ===== PWM_B34 构造（3 bytes, B通道波形，注意：写入A通道数据） =====
    static byte[] buildPwmB34(int x, int y, int z);

    // ===== V2 → V3 转换 =====
    // V3频率 = convert(X + Y)
    // V3强度 = Z * 5
    static int convertV2FreqToV3(int x, int y);
    static int convertV2StrengthToV3(int z);
}
```

> **V2 命名交叉陷阱**：PWM_A34 实际写入的是 **B 通道**波形数据，PWM_B34 写入的是 **A 通道**波形数据。代码中必须注释清楚。

## 6. V3 适配器实现细节

### 6.1 CoyoteV3Adapter 内部架构

```
CoyoteV3Adapter (public, 实现 DeviceProtocolAdapter)
  │
  ├── 字段
  │     ├── BluetoothGatt gatt              // BLE 连接
  │     ├── BluetoothGattCharacteristic writeChar   // 0x150A
  │     ├── BluetoothGattCharacteristic notifyChar  // 0x150B
  │     ├── int strengthA, strengthB        // 设备当前强度
  │     ├── int pendingSeqNo                // 流控：待确认的序列号
  │     ├── boolean waitingConfirm          // 流控：是否在等待 B1
  │     ├── ScheduledExecutorService timer  // 100ms 定时器
  │     └── 最新快照数据（volatile）
  │
  ├── connect()
  │     1. BluetoothAdapter.getRemoteDevice(address)
  │     2. device.connectGatt(context, false, gattCallback)
  │     3. onConnectionStateChange → CONNECTED
  │     4. discoverServices()
  │     5. onServicesDiscovered
  │        → 获取 0x150A (Write) + 0x150B (Notify)
  │        → setCharacteristicNotification(0x150B, true)
  │        → writeCharacteristic(0x150A, buildBF(默认值))
  │        → startTimer(100ms)
  │        → status.onStateChanged(CONNECTED)
  │
  ├── 100ms 定时器回调
  │     1. 读取最新快照（updateSnapshot 推送的数据）
  │     2. 调用 CoyoteV3Protocol.splitToSlots() 拆分波形
  │     3. 根据 waitingConfirm 决定强度解读方式
  │     4. 调用 CoyoteV3Protocol.buildB0() 构造指令
  │     5. gatt.writeCharacteristic(writeChar, cmd)
  │     6. status.onCycleStats()
  │
  ├── onCharacteristicChanged (0x150B Notify)
  │     → 数据头 0xB1 → CoyoteV3Protocol.parseB1()
  │     → 更新 strengthA/strengthB
  │     → 匹配 pendingSeqNo → 清除 waitingConfirm
  │
  ├── emergencyStop()
  │     → writeCharacteristic(buildEmergencyStop())
  │
  ├── flush()
  │     → 清空内部波形缓冲
  │     → 立即触发一次 B0 发送
  │
  └── release()
        → timer.shutdown()
        → gatt.disconnect() + close()
```

### 6.2 序列号流控状态机

```
                    ┌──────────────────┐
                    │  IDLE            │ ← waitingConfirm = false
                    │  可发送强度修改    │
                    └────────┬─────────┘
                             │ 发送 B0(seqNo≠0, 修改强度)
                             ▼
                    ┌──────────────────┐
                    │  WAITING_CONFIRM │ ← waitingConfirm = true
                    │  强度解读=不变    │
                    │  仅更新波形数据    │
                    └────────┬─────────┘
                             │ 收到 B1(seqNo==pendingSeqNo)
                             ▼
                    ┌──────────────────┐
                    │  IDLE            │
                    └──────────────────┘
```

### 6.3 重连逻辑

```
连接断开 (onConnectionStateChange DISCONNECTED)
  │
  ├── 音频和时间轴引擎不中断
  ├── 100ms 定时器继续运行（但 writeCharacteristic 静默失败，丢弃 B0）
  ├── 启动重连（最多 3 次，间隔 1s/3s/5s）
  │     ├── 成功 → flush() 同步当前波形 → status.onStateChanged(CONNECTED)
  │     └── 3 次均失败 → status.onFatalError() → 上层销毁适配器
  └── status.onStateChanged(RETRYING)
```

## 7. V2 适配器实现细节

### 7.1 CoyoteV2Adapter 与 V3 的差异

| 方面 | V3 | V2 |
|------|:--:|:--:|
| UUID 基地址 | `0000xxxx-0000-1000-8000-00805f9b34fb` | `955axxxx-0fe2-f5aa-a094-84b8d4f3e8ad` |
| 特性数量 | 2 个（Write + Notify） | 3 个（PWM_AB2 + PWM_A34 + PWM_B34） |
| 指令合并 | B0 合并双通道+波形 | 强度和波形分离写入 |
| 大小端 | 不需要 | 需要转换 |
| 强度范围 | 0-200 | 0-2047 |
| 波形强度 | 0-100 | Z值 0-31 |
| 反馈机制 | B1 消息 + 序列号 | 无（仅 Battery Level） |
| 软上限 | BF 指令 | 无 |

### 7.2 V2 输出窗口

V2 同样是 100ms 输出窗口，需要每 100ms 持续写入 PWM_A34/PWM_B34 维持波形。通道强度（PWM_AB2）写入后保持，无需周期发送。

## 附录 A：协议快速参考

### A.1 V3 B0 指令字节布局

```
偏移  长度  字段
0     1B    HEAD = 0xB0
1     1B    高4bit=序列号 | 低4bit=强度解读(bit3-2=A, bit1-0=B)
2     1B    A通道强度设定值 (0-200)
3     1B    B通道强度设定值 (0-200)
4-7   4B    A通道波形频率×4 (10-240)
8-11  4B    A通道波形强度×4 (0-100)
12-15 4B    B通道波形频率×4 (10-240)
16-19 4B    B通道波形强度×4 (0-100)
```

### A.2 强度解读方式

| bit3-2 (A) | bit1-0 (B) | 含义 |
|:---:|:---:|------|
| 00 | 00 | 不变 |
| 01 | 01 | 相对增加 |
| 10 | 10 | 相对减少 |
| 11 | 11 | 绝对设置 |

### A.3 频率换算速查

| 目标频率 | 波形频率(ms) | 协议值 |
|:---:|:---:|:---:|
| 100Hz | 10 | 10 |
| 50Hz | 20 | 20 |
| 20Hz | 50 | 50 |
| 10Hz | 100 | 100 |
| 5Hz | 200 | 120 |
| 1Hz | 1000 | 240 |

### A.4 V2 PWM 数据位布局

**PWM_AB2（3 bytes, 24 bits）：**
```
bits 23-22: 保留
bits 21-11: A通道强度 (0-2047)
bits 10-0:  B通道强度 (0-2047)
```

**PWM_A34 / PWM_B34（3 bytes, 24 bits）：**
```
bits 23-20: 保留
bits 19-15: Z值 (5 bits, 0-31) — 脉冲宽度，实际宽度 = Z × 5μs
bits 14-5:  Y值 (10 bits, 0-1023) — 脉冲间隔
bits 4-0:   X值 (5 bits, 0-31) — 连续脉冲数
```

> V2 数据需要大小端转换。PWM_A34 写入 B 通道波形，PWM_B34 写入 A 通道波形（命名交叉）。
