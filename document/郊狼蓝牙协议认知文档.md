## 1. 设备概述

### 1.1 郊狼情趣脉冲主机

郊狼是 DG-LAB 出品的情趣脉冲主机，内置**两组独立的脉冲生成模块**，分别对应 **A 通道**和 **B 通道**。每个通道可以独立控制强度和波形输出。

### 1.2 协议版本

| 版本 | 蓝牙名称 | 基础 UUID 格式 |
|------|---------|---------------|
| V2 (脉冲主机 2.0) | `D-LAB ESTIM01` | `955Axxxx-0FE2-F5AA-A094-84B8D4F3E8AD` |
| V3 (脉冲主机 3.0) | `47L121000` | `0000xxxx-0000-1000-8000-00805f9b34fb` |

### 1.3 核心概念

每个脉冲生成模块由以下要素控制：

- **通道强度**：控制脉冲电压（体感强度），范围 0-200（V3）或 0-2047（V2）
- **波形频率**：控制脉冲出现的节奏/频率
- **波形强度（脉冲宽度）**：控制单个脉冲的宽度，越宽感受越强
- **输出窗口**：V2 为 100ms，V3 为 25ms×4=100ms

## 2. V2 协议详解

### 2.1 蓝牙特性表

| 服务 UUID | 特性 UUID | 属性 | 名称 | 大小 | 说明 |
|:---------:|:---------:|:----:|:----:|:----:|------|
| 0x180A | 0x1500 | 读/通知 | Battery_Level | 1 字节 | 电量 (0-100) |
| 0x180B | 0x1504 | 读/写/通知 | PWM_AB2 | 3 字节 | AB 两通道强度 |
| 0x180B | 0x1505 | 读/写 | PWM_A34 | 3 字节 | A 通道波形数据 |
| 0x180B | 0x1506 | 读/写 | PWM_B34 | 3 字节 | B 通道波形数据 |

### 2.2 数据位布局

#### PWM_AB2 — 通道强度 (3 字节, 24 bits)

```
bits 23-22: 保留
bits 21-11: A 通道实际强度 (0-2047)
bits 10-0:  B 通道实际强度 (0-2047)
```

- 强度 S 范围：**0-2047**（不能超过 2047）
- APP 中每增加 1 点强度 = 实际写入值增加 7
- 写入后通道强度立刻改变并保持

#### PWM_A34 / PWM_B34 — 波形数据 (3 字节, 24 bits)

```
bits 23-20: 保留
bits 19-15: Z 值 (5 bits, 范围 0-31) — 脉冲宽度控制
bits 14-5:  Y 值 (10 bits, 范围 0-1023) — 脉冲间隔
bits 4-0:   X 值 (5 bits, 范围 0-31) — 连续脉冲数
```

> **注意**：PWM_A34 写入的是 **B 通道**波形数据，PWM_B34 写入的是 **A 通道**波形数据（命名与功能交叉）。
> V2 数据需要做**大小端转换**。

### 2.3 参数详解

#### X — 连续脉冲数

- 范围：0-31
- 含义：连续 X 毫秒发出 X 个脉冲

#### Y — 脉冲间隔

- 范围：0-1023
- 含义：X 个脉冲过后间隔 Y 毫秒再发出 X 个脉冲（循环）

#### Z — 脉冲宽度

- 范围：0-31
- 实际脉冲宽度 = Z × 5 μs
- 当 Z=20 时，脉冲宽度 = 100μs
- Z > 20 时更容易引起刺痛感

#### Frequency（频率特征值）

```
Frequency = X + Y
脉冲真实频率 = Frequency / 1000 (Hz)
```

取值范围 10~1000。X、Y 的最佳比例公式：

```
X = ((Frequency / 1000) ^ 0.5) × 15
Y = Frequency - X
```

若 X:Y 比例大于 1:9（如 [8,2]），波形整体感受会变弱。

### 2.4 输出窗口

每组 [X,Y,Z] 参数仅在 **0.1 秒（100ms）** 内有效，之后设备停止输出。因此需要**每 100ms 持续写入**以维持波形输出。

## 3. V3 协议详解

### 3.1 蓝牙特性表

| 服务 UUID | 特性 UUID | 属性 | 名称 | 大小 | 说明 |
|:---------:|:---------:|:----:|:----:|:----:|------|
| 0x180C | 0x150A | 写 | WRITE | 最长 20 字节 | 所有指令写入 |
| 0x180C | 0x150B | 通知 | NOTIFY | 最长 20 字节 | 所有回应消息返回 |
| 0x180A | 0x1500 | 读/通知 | READ/NOTIFY | 1 字节 | 电量信息 |

> 基础 UUID：`0000xxxx-0000-1000-8000-00805f9b34fb`  
> V3 数据**无需大小端转换**。

### 3.2 B0 指令 — 通道强度 + 波形数据

B0 是 V3 的核心指令，将两通道的强度和波形数据合并到一条 20 字节指令中，**每 100ms 写入一次**。

```
0xB0 (1 byte HEAD)
+ 序列号 (4 bits) + 强度值解读方式 (4 bits)
+ A 通道强度设定值 (1 byte)
+ B 通道强度设定值 (1 byte)
+ A 通道波形频率 × 4 (4 bytes)
+ A 通道波形强度 × 4 (4 bytes)
+ B 通道波形频率 × 4 (4 bytes)
+ B 通道波形强度 × 4 (4 bytes)
= 20 bytes 总计
```

#### 序列号 (4 bits)

- 范围：0b0000 ~ 0b1111
- 设为 0 时，设备不返回强度确认（B1 消息）
- 设为 > 0 时，设备通过 B1 消息以相同序列号返回修改后的通道强度
- **建议**：修改强度且序列号不为 0 时，等待 B1 返回相同序列号后再进行下一次强度修改

#### 强度值解读方式 (4 bits)

| 高 2 bits (A 通道) | 低 2 bits (B 通道) | 含义 |
|:---:|:---:|------|
| 00 | 00 | 对应通道强度**不做改变**（设定值无效） |
| 01 | 01 | **相对增加**：强度 += 设定值 |
| 10 | 10 | **相对减少**：强度 -= 设定值 |
| 11 | 11 | **绝对设置**：强度 = 设定值 |

**示例**：当前 A=10, B=10

| 解读方式 | A 设定值 | B 设定值 | 结果 A | 结果 B |
|:---:|:---:|:---:|:---:|:---:|
| 0b0000 | 5 | 8 | 10 | 10 |
| 0b0100 | 5 | 8 | 15 | 10 |
| 0b0010 | 5 | 8 | 10 | 2 |
| 0b0011 | 5 | 8 | 10 | 8 |
| 0b1101 | 5 | 8 | 5 | 18 |

#### 通道强度设定值

- 长度：1 byte
- 有效范围：**0-200**
- 超出范围的值以 0 处理
- 结合解读方式可实现增减或绝对值设定

#### 通道波形频率 (4 bytes)

- 每 byte 代表 **25ms 的波形频率**，4 bytes = 100ms
- 有效范围：**10-240**（最终发送值）
- 用户输入值范围 10~1000 需要经过换算：

```
输入值 10..100    → 输出值 = 输入值
输入值 101..600   → 输出值 = (输入值 - 100) / 5 + 100
输入值 601..1000  → 输出值 = (输入值 - 600) / 10 + 200
其他              → 输出值 = 10
```

#### 通道波形强度 (4 bytes)

- 每 byte 代表 **25ms 的波形强度**，4 bytes = 100ms
- 有效范围：**0-100**
- 若某通道任一值超出范围，该通道**全部 4 组数据被放弃**
- **Tip**：若只想对单通道输出波形，将另一通道的某个强度值设为 >100（如 101），该通道数据即被放弃

### 3.3 BF 指令 — 软上限与平衡参数

BF 指令设置通道强度软上限和波形平衡参数，**断电保存**。

```
0xBF (1 byte HEAD)
+ AB 两通道强度软上限 (2 bytes)
+ AB 两通道波形频率平衡参数 (2 bytes)
+ AB 两通道波形强度平衡参数 (2 bytes)
= 7 bytes 总计
```

> **重要**：BF 指令写入后直接生效，无返回值。**每次重连设备后必须重新写入 BF 指令**，防止出现意外的软上限值。

#### 通道强度软上限

- 每通道 1 byte，范围 0-200
- 限制通道强度能达到的最大值（如设 A=150，则 A 通道强度 ≤ 150）
- 超出范围的值不会修改软上限

#### 频率平衡参数 1

- 每通道 1 byte，范围 0-255
- 调整不同频率波形的相对体感强度
- 值越大，低频波形冲击感越强

#### 频率平衡参数 2

- 每通道 1 byte，范围 0-255
- 调整波形脉冲宽度
- 值越大，低频波形刺激越强

### 3.4 蓝牙回应消息

#### B1 消息 — 强度变化反馈

```
0xB1 (1 byte HEAD)
+ 序列号 (1 byte)
+ A 通道当前实际强度 (1 byte)
+ B 通道当前实际强度 (1 byte)
= 4 bytes 总计
```

- 当通道强度发生变化时（B0 指令、物理滚轮、APP 调节）立刻上报
- 由 B0 引起的强度变化：序列号与 B0 中一致
- 其他原因引起的强度变化：序列号 = 0

### 3.5 强度控制伪代码

```
// 状态变量
isInputAllowed = true           // 当前是否允许输入强度
accumulatedStrengthValueA = 0   // A 通道未写入的累计强度变化值
deviceStrengthValueA = 0        // 设备当前 A 通道强度值
orderNo = 0                     // 序列号
inputOrderNo = 0                // B0 写入的序列号

// 每次 B0 写入前调用
fun strengthDataProcessingA():
    if isInputAllowed:
        strengthParsingMethod = if accumulatedStrengthValueA > 0 → 0b0100
                                else if accumulatedStrengthValueA < 0 → 0b1000
                                else → 0b0000
        orderNo += 1
        inputOrderNo = orderNo
        isInputAllowed = false
        strengthSettingValueA = abs(accumulatedStrengthValueA)
        accumulatedStrengthValueA = 0
    else:
        orderNo = 0
        strengthParsingMethod = 0b0000
        strengthSettingValueA = 0

// B1 消息回调
fun strengthDataCallback(returnOrderNo, returnStrengthValueA):
    deviceStrengthValueA = returnStrengthValueA
    if returnOrderNo == inputOrderNo:
        isInputAllowed = true
        strengthParsingMethod = 0b0000
        strengthSettingValueA = 0
        inputOrderNo = 0

// 强度归零
fun strengthZero():
    strengthParsingMethod = 0b1100
    strengthSettingValueA = 0
    orderNo = 1
    inputOrderNo = orderNo
```

## 4. 波形概念详解

### 4.1 波形频率

郊狼将每秒分为 1000 毫秒，每毫秒可产生一次脉冲。**波形频率**表示一个输出单元的时长（由 X 个脉冲 + Y 个间隔组成），单位 ms。

| 脉冲频率 | 波形频率 |
|:------:|:------:|
| 1Hz | 1000ms |
| 5Hz | 200ms |
| 10Hz | 100ms |
| 50Hz | 20ms |
| 100Hz | 10ms |
| 500Hz | 2ms |
| 1000Hz | 1ms |

> 例：参数 [1,9] 表示每 9ms 发出 1 个脉冲，总耗时 10ms，即脉冲频率 100Hz。参数 [5,95] 表示每 95ms 发出 5 个连续脉冲，总耗时 100ms，体感上 5 个脉冲合成一次感受，等效 10Hz。

### 4.2 V3 波形频率换算表

由于人体对频率细微变化不敏感，V3 对波形频率做了压缩转换：

| 波形频率 (ms) | 输出值 | 脉冲频率 |
|:-----:|:---:|:------:|
| 10ms | 10 | 100Hz |
| 20ms | 20 | 50Hz |
| 50ms | 50 | 20Hz |
| 100ms | 100 | 10Hz |
| 110ms | 102 | 9Hz |
| 150ms | 110 | 6.6Hz |
| 650ms | 204 | 1.53Hz |
| 680ms | 208 | 1.47Hz |
| 750ms | 215 | 1.33Hz |

### 4.3 波形强度（脉冲宽度）

- 一次脉冲由两个对称的正负单极性脉冲组成
- 脉冲高度（电压）由通道强度决定
- **脉冲宽度**控制感受强弱：越宽感受越强
- V2 强度范围：**0-20**（Z 值 0-31，实际宽度 Z×5μs）
- V3 强度范围：**0-100**
- 映射关系：V2 强度 20 ≈ V3 强度 100

### 4.4 输出窗口与频率冲突

- V2 输出窗口：100ms
- V3 输出窗口：25ms × 4 组 = 100ms

当波形频率时长大于输出窗口时，若下一周期频率发生变化，实际输出效果经过设备内部复杂处理，可能与输入值不完全对应。

**建议**：
1. 若希望稳定输出某频率，持续输入相同的波形频率值
2. 波形频率值大于输出窗口时避免频繁变化

## 5. V2 与 V3 对比与转换

### 5.1 主要差异对比

| 特性 | V2 | V3 |
|------|:--:|:--:|
| 蓝牙名称 | D-LAB ESTIM01 | 47L121000 |
| 基础 UUID | 955A... | 0000... |
| 指令结构 | 分离的 PWM_AB2/A34/B34 | 合并的 B0 指令 |
| 大小端转换 | 需要 | 不需要 |
| 强度范围 | 0-2047 (S 值) | 0-200 |
| 波形强度范围 | 0-20 (Z 值) | 0-100 |
| 波形频率输入 | X+Y 原始值 | 压缩转换值 (10-240) |
| 强度修改方式 | 直接写入绝对值 | 支持相对增减/绝对设置 |
| 强度反馈 | 无序列号机制 | B1 消息 + 序列号确认 |
| 软上限 | 无 | BF 指令支持 |
| 输出窗口 | 100ms (1 组) | 25ms×4 组 (100ms) |

### 5.2 V2 → V3 转换公式

```
V3 波形频率 = V2 (X + Y)，再执行 10~1000 → 10~240 的压缩转换
V3 波形强度 = V2 Z × 5
```

## 7. 与 HypnoVibe 的集成要点

### 7.1 子包封装架构

在 HypnoVibe 中，郊狼协议作为**完整自治的子包**封装在 `infrastructure/ble/adapter/coyote/` 下：

```
infrastructure/ble/adapter/coyote/    ← 郊狼协议子包（完整封装）
  ├── CoyoteV3Adapter.java            ← public, 实现 DeviceProtocolAdapter（V3 设备）
  ├── CoyoteV2Adapter.java            ← public, 实现 DeviceProtocolAdapter（V2 设备）
  └── (协议编解码、数据模型、常量等内部类均为 package-private)
```

**封装原则：**
- 对外只暴露 `CoyoteV3Adapter` / `CoyoteV2Adapter` 两个 public 类
- 子包内部自主管理：协议编解码、BLE 连接、100ms 定时器、流控状态机、频率换算等
- 架构层通过统一的 `DeviceProtocolAdapter` 接口与子包交互，不感知内部实现
- 详细实现方案参见 [郊狼协议移植方案.md](郊狼协议移植方案.md)

### 7.2 关键集成点

| 集成点 | 说明 |
|--------|------|
| **BLE 写入周期** | 子包内部维护 100ms 定时器，持续发送波形指令维持输出 |
| **BF 指令初始化** | 子包在连接成功后自动写入 BF 指令（软上限+平衡参数） |
| **Notify 绑定** | 子包内部绑定 0x150B 特性 Notify，接收并处理 B1 强度反馈 |
| **强度映射** | V3 通道强度 0-200，V2 通道强度 0-2047，由子包内部处理 |
| **波形频率换算** | 用户输入 10-1000ms → 协议值 10-240，由子包内部换算 |
| **波形强度映射** | V3 波形强度 0-100，V2 Z 值 0-31，由子包内部处理 |
| **安全停止** | 通过 `DeviceProtocolAdapter.emergencyStop()` 触发，子包内部归零 |
| **多设备支持** | 每个郊狼设备独立创建 Adapter 实例，各自维护 BLE 连接 |
| **V2/V3 自动识别** | 根据蓝牙名称区分版本，加载对应 Adapter |

### 7.3 子包内部处理流程（参考）

以下流程由郊狼子包内部自主实现，架构层无需关心：

```
每 100ms 周期（子包内部定时器驱动）:
1. 获取当前各通道目标强度（来自时间轴引擎推送的快照）
2. 确定强度值解读方式（相对增减 or 绝对设置）
3. 获取当前各通道波形数据（频率 × 4 + 强度 × 4）
4. 拼接 20 字节 B0 指令
5. 通过 BLE GATT writeCharacteristic(0x150A, data) 发送
6. 如有序列号，等待 B1 回应确认
```

### 7.4 波形文件转换

HypnoVibe 的波形文件系统需要支持（由后续 Phase 实现，子包提供协议层支持）：

- **DG-LAB V2 波形导入**：解析 X/Y/Z 参数 → 转换为统一波形格式
- **DG-LAB V3 波形导入**：解析频率/强度序列 → 转换为统一波形格式
- **反向输出**：统一波形格式 → B0 指令所需的频率+强度数据（由子包内部处理）

### 7.5 注意事项

1. **时序要求严格**：100ms 周期不可中断，子包内部使用高优先级定时器驱动
2. **单通道输出**：只需单通道时，将另一通道的某个强度值设为 >100
3. **强度修改确认**：需要精确控制强度时，启用序列号机制等待 B1 确认
4. **V2/V3 双版本**：子包提供独立的 V2/V3 适配器，各自处理不同的 UUID、指令格式、大小端转换
5. **首次连接**：连接后必须先发 BF 指令，再开始 B0 循环（由子包自动处理）
